# USM Notes — WordPress Plugin

**Лабораторная работа №4:** Разработка плагина для WordPress  
**Автор:** Кристева Ирина  
**Дисциплина:** Системы управления контентом (CMS)

---

## 📋 Описание лабораторной работы

Цель: изучить расширяемую модель данных WordPress — создать Custom Post Type (CPT), пользовательскую таксономию, метаданные с метабоксом в панели администратора, а также реализовать шорткод для отображения данных на сайте.

Плагин **USM Notes** добавляет на сайт раздел «Заметки» с приоритетами и датой напоминания.

---

## 🚀 Инструкции по запуску

### Требования
- WordPress 5.8+
- PHP 7.4+
- Локальная среда: [Local by Flywheel](https://localwp.com/), XAMPP, Laragon и др.

### Установка плагина

1. Скопируйте папку `usm-notes` в директорию:
   ```
   wp-content/plugins/usm-notes/
   ```

2. Убедитесь, что структура файлов такова:
   ```
   usm-notes/
   ├── usm-notes.php   ← основной файл плагина
   └── usm-notes.css   ← стили для фронтенда
   ```

3. Активируйте режим отладки в `wp-config.php`:
   ```php
   define('WP_DEBUG', true);
   ```

4. Войдите в панель администратора WordPress → **Плагины** → найдите **USM Notes** → нажмите **Активировать**.

5. После активации в меню появится раздел **Заметки** с таксономией **Приоритет**.

## 📖 Документация плагина

### Что делает плагин

| Компонент | Описание |
|-----------|----------|
| CPT `usm_note` | Пользовательский тип записи «Заметки» |
| Таксономия `usm_priority` | Иерархическая таксономия «Приоритет» (High / Medium / Low) |
| Метабокс «Дата напоминания» | Поле `date` в редакторе заметки с валидацией |
| Шорткод `[usm_notes]` | Вывод списка заметок с фильтрами на любой странице |

---

### Шорткод `[usm_notes]`

Вставьте шорткод на любую страницу сайта.

#### Атрибуты

| Атрибут | Тип | Описание |
|---------|-----|----------|
| `priority` | string | Фильтр по slug приоритета (`high`, `medium`, `low`) |
| `before_date` | string | Фильтр: заметки с датой напоминания ≤ указанной (`YYYY-MM-DD`) |

#### Примеры использования

```
[usm_notes]
```
Выводит **все** опубликованные заметки.

```
[usm_notes priority="high"]
```
Выводит только заметки с приоритетом **High**.

```
[usm_notes before_date="2025-04-30"]
```
Выводит заметки, у которых дата напоминания **до 30 апреля 2025**.

```
[usm_notes priority="medium" before_date="2025-06-01"]
```
Комбинированный фильтр: приоритет Medium **и** дата до 1 июня 2025.

Если заметки по фильтру не найдены, выводится сообщение:  
> *Нет заметок с указанными параметрами.*

---

### Валидация метабокса

- Поле **«Дата напоминания»** обязательно для заполнения.
- Дата **не может быть в прошлом** — при попытке сохранить прошедшую дату появится уведомление в панели администратора.
- При сохранении используется **nonce** для защиты от CSRF-атак.

## ❓ Ответы на контрольные вопросы

### 1. В чём принципиальное отличие пользовательской таксономии от метаполя?

**Таксономия** — это способ классификации записей по общим признакам. Термины таксономии — самостоятельные объекты в базе данных, ими можно управлять централизованно (создавать, редактировать, удалять), они поддерживают иерархию и архивные страницы.

**Метаполе** — произвольное значение, прикреплённое к конкретной записи. Оно уникально для каждого поста и не имеет своей страницы архива.

**Когда использовать таксономию:**  
Когда нужна классификация, общая для многих записей, и пользователи должны иметь возможность фильтровать записи по этому признаку через URL. Пример: **приоритет заметки** (High / Medium / Low) — разные заметки используют одни и те же термины, можно создать страницу `/priority/high/` со всеми срочными заметками.

**Когда использовать метаполе:**  
Когда значение уникально для каждой записи. Пример: **дата напоминания** — у каждой заметки своя дата, общей страницы архива не нужно, данные носят фактический, а не классификационный характер.

---

### 2. Зачем нужен nonce при сохранении метаполей и что будет без него?

**Nonce** (Number Used Once) — это одноразовый токен безопасности, который WordPress генерирует для конкретного действия и пользователя.

При сохранении метабокса:
1. При рендеринге формы генерируется nonce: `wp_nonce_field('usm_save_due_date', 'usm_due_date_nonce')`.
2. При сохранении нonce проверяется: `wp_verify_nonce($_POST['usm_due_date_nonce'], 'usm_save_due_date')`.

**Без nonce** сайт уязвим к **CSRF-атаке** (Cross-Site Request Forgery): злоумышленник может создать страницу с формой, которая отправит POST-запрос на сайт жертвы от её имени. Поскольку браузер автоматически отправляет cookies сессии, WordPress не отличит легитимный запрос от поддельного. В результате атакующий может изменить метаданные любой записи без ведома владельца сайта.

### 3. Важнейшие параметры `register_post_type()` и `register_taxonomy()` для фронтенда и UX

**Для `register_post_type()`:**

| Параметр | Влияние на фронтенд и UX |
|----------|--------------------------|
| `public => true` | CPT доступен на фронтенде: появляются архивные страницы и страницы одиночных записей, которые можно просматривать в браузере. |
| `has_archive => true` | Создаёт страницу архива `/notes/` — пользователи могут видеть все заметки по одному URL без шорткода. |
| `menu_icon` | Задаёт иконку в меню администратора (используем `dashicons-sticky`), что улучшает навигацию и визуальное восприятие. |
| `labels` | Все подписи кнопок, заголовков и сообщений в интерфейсе WP Admin. Без них используются стандартные «Post»/«Posts», что запутывает пользователя. |
| `show_in_rest => true` | Включает поддержку REST API и Gutenberg-редактора, что критично для современного UX редактирования. |

**Для `register_taxonomy()`:**

| Параметр | Влияние на фронтенд и UX |
|----------|--------------------------|
| `hierarchical => true` | Таксономия работает как категории (с чекбоксами в редакторе), а не как теги — удобнее для выбора фиксированных приоритетов. |
| `public => true` | Термины таксономии получают свои URL-адреса (`/priority/high/`), что позволяет фильтровать заметки на фронтенде. |
| `rewrite => ['slug' => 'priority']` | Задаёт читаемый ЧПУ-URL для архива таксономии, улучшает SEO и понятность ссылок. |

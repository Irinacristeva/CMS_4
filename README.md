## Шаг 3. Регистрация Custom Post Type (CPT)

Добавлена функция регистрации CPT «Заметки» с помощью `register_post_type()`:

```php
function usm_register_notes_cpt() {
    $labels = array(
        'name'          => 'Заметки',
        'singular_name' => 'Заметка',
        'menu_name'     => 'Заметки',
        'add_new'       => 'Добавить заметку',
        'add_new_item'  => 'Новая заметка',
        'edit_item'     => 'Редактировать заметку',
        'all_items'     => 'Все заметки',
        'not_found'     => 'Заметки не найдены',
    );
    $args = array(
        'labels'       => $labels,
        'public'       => true,
        'has_archive'  => true,
        'rewrite'      => array('slug' => 'notes'),
        'supports'     => array('title', 'editor', 'author', 'thumbnail'),
        'menu_icon'    => 'dashicons-clipboard',
        'show_in_rest' => true,
    );
    register_post_type('usm_note', $args);
}
add_action('init', 'usm_register_notes_cpt');
```

**Параметры CPT:**
- `public => true` — тип виден и в админке, и на сайте
- `has_archive => true` — создаётся архивная страница `/notes/`
- `supports` — включены: заголовок, редактор, автор, миниатюра
- `menu_icon` — иконка в меню админки из библиотеки Dashicons
- Хук `init` — регистрация CPT при каждой загрузке WordPress

После сохранения в левом меню админки появился раздел «Заметки».

## Шаг 4. Регистрация пользовательской таксономии

Добавлена функция регистрации таксономии «Приоритет» с помощью `register_taxonomy()`:

```php
function usm_register_priority_taxonomy() {
    $labels = array(
        'name'          => 'Приоритеты',
        'singular_name' => 'Приоритет',
        'menu_name'     => 'Приоритеты',
        'all_items'     => 'Все приоритеты',
        'add_new_item'  => 'Добавить приоритет',
    );
    $args = array(
        'labels'            => $labels,
        'hierarchical'      => true,
        'public'            => true,
        'show_admin_column' => true,
        'rewrite'           => array('slug' => 'priority'),
        'show_in_rest'      => true,
    );
    register_taxonomy('usm_priority', 'usm_note', $args);
}
add_action('init', 'usm_register_priority_taxonomy');
```

**Параметры таксономии:**
- `hierarchical => true` — работает как рубрики (иерархически)
- `public => true` — видна на сайте
- `show_admin_column => true` — колонка «Приоритет» в списке заметок
- Таксономия привязана к CPT `usm_note`

Через **Заметки → Приоритеты** добавлены три термина:
- **High** (ярлык: `high`)
- **Medium** (ярлык: `medium`)
- **Low** (ярлык: `low`)

## Шаг 6. Создание шорткода для отображения заметок

Создан шорткод `[usm_notes]` с двумя необязательными параметрами:
- `priority` — фильтр по приоритету (`high`, `medium`, `low`)
- `before_date` — фильтр по дате напоминания (`YYYY-MM-DD`)

```php
function usm_notes_shortcode($atts) {
    $atts = shortcode_atts(array(
        'priority'    => '',
        'before_date' => '',
    ), $atts, 'usm_notes');
    // WP_Query с фильтрами по таксономии и метаполю
}
add_shortcode('usm_notes', 'usm_notes_shortcode');
```

**Логика работы:**
- Если `priority` не указан — показываются все приоритеты
- Если `before_date` не указана — показываются все даты
- Если заметок нет — выводится сообщение «Нет заметок с заданными параметрами»

Добавлены CSS-стили через `wp_add_inline_style` — карточки заметок отображаются в сетке 2 колонки. Цвет левой полосы карточки меняется в зависимости от приоритета: красный (High), оранжевый (Medium), зелёный (Low).

## Контрольные вопросы

### 1. Чем пользовательская таксономия отличается от метаполя?

**Таксономия** — это способ группировать записи по заранее заданным категориям. Термины таксономии создаются один раз и назначаются множеству записей. WordPress автоматически создаёт архивные страницы для каждого термина (например, `/priority/high/`), поддерживает фильтрацию и отображение в виде облака меток или списка рубрик.

**Метаполе** — это произвольные данные, уникальные для каждой конкретной записи. Они не создают архивных страниц и не предполагают выбор из готового списка.

**Когда выбрать таксономию:**  
Когда значения повторяются у многих записей и их количество ограничено. Например: приоритет заметки (High/Medium/Low), жанр книги, рубрика статьи. В данной работе приоритет реализован как таксономия — три фиксированных значения повторяются у разных заметок.

**Когда выбрать метаполе:**  
Когда данные уникальны для каждой записи и не являются категорией. Например: дата напоминания, цена товара, номер телефона. В данной работе дата напоминания реализована как метаполе — у каждой заметки своя уникальная дата.

---

### 2. Зачем нужен nonce при сохранении метаполей?

**Nonce** (Number Used Once) — это одноразовый токен безопасности, который WordPress генерирует для каждой формы.

**Проблема без nonce:** злоумышленник может создать поддельную форму на стороннем сайте и заставить браузер авторизованного администратора отправить запрос на сохранение данных в WordPress. Это называется **CSRF-атака** (Cross-Site Request Forgery).

**Как работает защита:**
1. При отрисовке формы WordPress генерирует токен: `wp_nonce_field('usm_save_due_date', 'usm_due_date_nonce')`
2. Токен скрыто вставляется в форму
3. При сохранении проверяем токен: `wp_verify_nonce($_POST['usm_due_date_nonce'], 'usm_save_due_date')`
4. Если токен не совпадает — прерываем выполнение

**Что произойдёт без проверки nonce:**  
Любой сторонний сайт сможет от имени авторизованного администратора отправлять запросы и изменять метаданные записей в базе данных.

---

### 3. Важные аргументы `register_post_type()` и `register_taxonomy()`

**1. `public => true`**  
Делает тип записей видимым одновременно в админке и на сайте. Если установить `false` — CPT будет существовать только внутри кода, пользователи не смогут просматривать записи на сайте. Это критически важно для любого публичного раздела.

**2. `has_archive => true`**  
Создаёт архивную страницу со списком всех записей (например, `/notes/`). Без этого аргумента у CPT не будет отдельной страницы на сайте — пользователи смогут открывать только отдельные записи, но не видеть их список. Важно для навигации и UX.

**3. `rewrite => array('slug' => '...')`**  
Задаёт красивый человекочитаемый URL для записей и архива. Без этого WordPress использует технический URL вида `?post_type=usm_note`, что неудобно для пользователей и плохо для SEO.

**4. `show_admin_column => true`** (для таксономий)  
Добавляет колонку таксономии в список записей в админке. Без этого администратор не видит приоритет заметки в общем списке и вынужден открывать каждую запись по отдельности. Важно для удобства работы в админ-панели.
